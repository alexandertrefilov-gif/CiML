# CImL Phase 4 – Test- und Abnahmeplan

## 1. Login und Sichtbarkeit

- [ ] Mitgliederbereich ist ohne Login nicht erreichbar.
- [ ] Interne Seiten haben noindex.
- [ ] Besucher sehen keine internen Termine.
- [ ] Besucher sehen keine Dienstpläne.
- [ ] Besucher sehen keine internen Dokumente.

## 2. Rollenprüfung

- [ ] Mitglied sieht nur member_only Inhalte.
- [ ] Teammitglied sieht nur eigene Team-Inhalte.
- [ ] Teamleiter sieht Inhalte des eigenen Teams.
- [ ] Leitung sieht bereichsübergreifende interne Inhalte.
- [ ] Administrator sieht technische Verwaltung.
- [ ] Deaktivierter Benutzer verliert Zugriff.

## 3. Dateien / Downloads

- [ ] Interne Dateien sind nicht über direkte öffentliche URL abrufbar.
- [ ] Download wird gegen Rolle/Team geprüft.
- [ ] Dokumentenzugriff kann protokolliert werden.
- [ ] Abgelaufene Dokumente werden markiert.

## 4. Dienstpläne

- [ ] Dienstplan ist nur für berechtigte Teams sichtbar.
- [ ] Status draft ist nicht für Teammitglieder sichtbar.
- [ ] published/confirmed wird korrekt angezeigt.
- [ ] Änderungen werden markiert.

## 5. Schulungen

- [ ] Schulungen sind nach Rolle/Team sichtbar.
- [ ] Abschlussstatus kann gespeichert werden.
- [ ] Pflichtschulungen werden hervorgehoben.
- [ ] Kinderschutz-Schulung nur für relevante Teams sichtbar.

## 6. Grenze zu Phase 3

- [ ] Keine Seelsorge-Anfragen im Mitgliederbereich.
- [ ] Keine vertraulichen Gebetsanliegen im Mitgliederbereich.
- [ ] Keine Kinderschutzfälle im Mitgliederbereich.
- [ ] Nur Schutzkonzept/Schulungsmaterial, keine Fallverwaltung.

## 7. Demo-Daten

- [ ] Demo-Daten installierbar.
- [ ] Demo-Daten klar markiert.
- [ ] Demo-Daten löschbar.
- [ ] Nach Löschung keine Demo-Mitglieder/Dienstpläne sichtbar.

## 8. Release-Gate

NO-GO, wenn interne Dokumente ohne Login abrufbar sind.  
NO-GO, wenn Phase-3-Daten im Mitgliederbereich sichtbar sind.
